HAPLab Metabolic Analyzer · User Guide

Getting Started

HAPLab Metabolic Analyzer is a web-based data reduction and reporting tool designed for exercise physiologists and researchers. It integrates Parvo TrueOne 2400 metabolic cart data with lactate measurements, allowing users to manually identify ventilatory and lactate thresholds through interactive visualization. The tool automatically calculates and reports physiological parameters at user-defined thresholds, generates training zones, and provides longitudinal comparison of pre- and post-intervention testing, eliminating the need for manual calculations and separate spreadsheets.

Supported File Formats

  • Raw Parvo TrueOne 2400 CSV export
  • Cleaned/reformatted CSV files
  • Saved JSON session files (.json)

Quick Start

  1. Load a CSV file using the Load Session button
  2. Review metabolic charts and auto-detected VT1/VT2 thresholds
  3. Adjust thresholds by dragging the blue (VT1) and red (VT2) dashed lines on any chart
  4. Enter lactate data in the Lactate Analysis table if collected during the test
  5. Save your session as JSON for future use via Save Session

Metabolic Charts

Chart Descriptions

  • RER vs Time: Respiratory Exchange Ratio over test duration. RER crossing ~1.0 indicates maximal carbohydrate oxidation.
  • VE/VO2 and VE/VCO2 vs HR: Ventilatory equivalents used for VT1/VT2 detection. VT1 = nadir of VE/VO2 without rise in VE/VCO2. VT2 = simultaneous rise in both.
  • FAT & CHO Oxidation vs HR: Substrate utilization rates. The crossover point indicates a metabolic shift toward carbohydrate dominance.
  • Minute Ventilation vs HR: VE inflection points correspond to ventilatory thresholds.
  • VO2/kg vs Time: Oxygen uptake progression. A plateau may indicate VO2max attainment.
  • VO2/kg vs HR (Cardiac Efficiency): Rightward shift between tests indicates improved cardiac efficiency at the same oxygen demand.

Adjusting VT1 / VT2

  • Drag the blue dashed line to set VT1 (Aerobic / First Ventilatory Threshold)
  • Drag the red dashed line to set VT2 (Anaerobic / Second Ventilatory Threshold)
  • All charts, tables, and training zones update instantly when lines are dragged
  • Click the expand icon (↗) in the corner of any chart to enlarge it
  • Use arrow keys or scroll to navigate between charts in expanded view

Lactate Analysis

Entering Lactate Data

  • Enter one lactate value per stage in the Lactate (mmol/L) column
  • Resting lactate can be entered separately in the field above the table
  • Blank stages are excluded from curve fitting
  • Speed units toggle (km/h / mph) available above the lactate charts

Polynomial Regression

  • App fits 2nd or 3rd degree polynomial to lactate data (toggle available)
  • R² value shown on each chart, values >0.95 indicate excellent fit
  • Poor fit (R² <0.92) may indicate outliers or insufficient data points (<4 stages recommended)

Dmax Methods

  • Traditional Dmax (Cheng 1992): maximum perpendicular distance from a reference line connecting the first and last lactate data points
  • Modified Dmax (Bishop 1998): same method but the reference line starts at the first point where lactate rises ≥0.4 mmol/L above baseline, making it more robust for athletes with flat early lactate curves
  • Enable Show Dmax Construction to visualize the geometric construction on the chart

Threshold Lactate Values

  • Polynomial: interpolated value from the fitted curve at the exact threshold position
  • Nearest Measured: actual collected lactate value from the closest stage to the threshold

Threshold Summary & Training Zones

Threshold Summary Table

  • VT1/VT2 values are interpolated from metabolic cart data at the exact threshold HR positions
  • Lactate values are shown only if lactate data was entered
  • All values update live when thresholds are dragged on any chart
  • VT = Ventilatory Threshold (metabolic cart); LT = Lactate Threshold (lactate curve)

Training Zones

  • 5-zone model derived from VT1, VT2, and peak HR from the test
  • Zone HR boundaries update automatically when thresholds are moved
  • CHO/FAT oxidation values averaged from metabolic cart data within each zone's HR range
  • Lactate ranges derived from the personal polynomial curve at each zone's HR boundaries
Training zones are estimates based on the GXT data. Individual response may differ in field conditions.

Longitudinal Comparison

Loading a Comparison Session

  • Click + Load Comparison and select a second CSV or JSON session file
  • Primary session shown in red (right panels)
  • Comparison session shown in gray (left panels)
  • A modality mismatch warning appears if the two sessions used different protocols

Adjusting Thresholds Independently

  • Each session's VT1/VT2 lines can be dragged independently on their respective charts
  • The Reset button restores original auto-detected positions
  • The comparison column in the longitudinal table updates when comparison thresholds are dragged

Longitudinal Comparison Table

  • Shows side-by-side metrics for both sessions
  • Green = improvement, Red = decline relative to the comparison session
  • Reference Speed: enter a fixed speed to compare HR efficiency between sessions at the same absolute workload

View Modes

  • Side by Side: each session in its own panel for direct visual comparison
  • Overlay: both sessions plotted on the same chart axes

Saving & Exporting

Save Session (JSON)

  • Saves all data, lactate entries, threshold positions, modality, and app settings
  • Load later via Load Session to resume exactly where you left off
  • Recommended after every testing session before closing the browser

Suggested File Naming

SubjectID_Date_Protocol.json
e.g. JDoe_20240315_GXT.json

Data Privacy

  • All data stays local in your browser, nothing is uploaded to any server
  • JSON session files contain all entered data including patient information
  • Handle files according to your IRB / institutional data privacy requirements
There is no cloud backup. Keep copies of JSON session files in a secure, backed-up location.

Tips

  • Do not modify column headers in raw Parvo exports, the app locates data by matching the exact header pattern.
  • Blank lactate cells are treated as missing, not zero; only filled cells are included in curve fitting.
  • Speed can be in MPH or km/h, a unit toggle is available in the Lactate Analysis section above the charts.
  • Save your session as JSON after entering lactate data to avoid re-entry next time.
